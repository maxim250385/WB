# WB / Ozon Product Extractor

Автоматически извлекает атрибуты товаров из текстов поставщиков и сохраняет в Excel.

## Структура проекта

```
├── main.py                  # Точка входа
├── config.py                # Константы, пути, модели Gemini
├── .env                     # API-ключи (НЕ коммитить в git!)
├── .env.example             # Шаблон .env
├── proxies.txt              # Прокси (ip:port:login:password)
├── requirements.txt
├── input/                   # Сюда кладёшь .txt с описаниями товаров
├── output/                  # Сюда падают готовые .xlsx
└── modules/
    ├── gemini_engine.py     # Gemini API: каскад моделей + прокси
    ├── prompts.py           # Промпты и схемы полей по категориям
    ├── category_detector.py # Определение категории товара
    ├── validator.py         # Валидация и пометка пустых полей
    └── excel_writer.py      # Запись в .xlsx с форматированием
```

## Быстрый старт

```bash
# 1. Установить зависимости
pip install -r requirements.txt

# 2. Создать .env с ключами
cp .env.example .env
# Открыть .env и вставить ключи Gemini (aistudio.google.com/apikey)
# Можно указать один или несколько через запятую: GEMINI_API_KEYS=ключ1,ключ2

# 3. Положить текст поставщика в input/
# (файл: input/товар1.txt)

# 4. Запустить
python main.py
```

## Пример выходных данных (Excel)

| Артикул поставщика | Наименование              | Категория | Бренд | Страна | Длина (см) | ... | Материал верха         |
|--------------------|---------------------------|-----------|-------|--------|-----------|-----|------------------------|
| NK-AIR-2024-BLK-42 | Кроссовки беговые Nike... | обувь     | Nike  | Вьетнам | 33       | ... | текстиль + синт. кожа  |

Ячейки с «Уточнить у поставщика» подсвечены оранжевым — это обязательные поля которые не нашлись в тексте.
